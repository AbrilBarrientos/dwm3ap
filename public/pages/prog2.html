Modelo Cliente - Servidor

Los archivos de una página están alojados en un servidor

Cada servidor tiene una dirección IP

Para acceder al contenido de la IP es necesario un CLIENTE (navegador web o browser)

* CLIENTE = NAVEGADOR

* Cualquier software con capacidad de realizar pedidos es considerado un Cliente

El Cliente se conecta mediante una dirección (IP o URL) y comienza la interacción con el servidor.

Protocolo HTTP / HTTPS

Hyper Text Transfer Protocol

  • Protocolo básico para transferir información a través de la web.
  • Cuando un sitio comienza con http:// significa que la información que se está transmitiendo entre el navegador (cliente) y el servidor NO ESTÁ CIFRADA.

Hyper Text Transfer Protocol Secure

  • Versión segura de HTTP.
  • La S significa "seguro"
  • La informaciín transmitida ESTÁ CIFRADA.
  • Asegura que los datos no puedan ser alterados ni leídos por nadie mientras están en tránsito.
  • Protege la privacidad de los datos.

Procesos Client-Side

  • Procesos en el navegador que ocurren del lado del cliente.
  • El navegador recibe el código y lo muestra en tiempo real.
  • El usuario puede verlo y modificarlo.
  • No necesita servidor para funcionar

Procesos Server-Side

  • Se ejecuta en el servidor web.
  • Procesa datos, maneja bases de datos y genera contenido dinámico.
  • El código permanece oculto.
  • SI necesita servidor para funcionar.

Los procesos SERVER-SIDE se llevan a cabo en el servidor ANTES de enviar la respuesta al cliente.

El cliente accede únicamente al resultado y nunca al código

Sitios web estáticos

  • Solo usan lenguajes client-side (HTML, CSS, JS)
  • El contenido es siempre el mismo para todos.
  • Son archivos fijos (no cambian solos) y en texto plano.

Sitios web dinámicos

  • Usan código que se ejecuta en el servidor.
  • El contenido se genera en el momento, no es fijo.

PHP - Hypertext Preprocessor

Lenguaje que genera HTML de forma automática (programado)

Se escribe HTML en el documento.php, pero se ejecuta en el servidor.

El navegador solo sabe interpretar HTML

El archivo principal suele ser index.php

Servidor Local

PHP necesita un servidor para funcionar porque es server-side.

Un servidor local simula un servidor real en la computadora.

Softwares de Servidor Local según sistema operativo

  • Windows
    • WAMP
  • Mac
    • MAMP
  • Linux
    • LAMP
  • Todos
    • XAMPP
